Cài đặt & Cấu hình JAMA HOME CRM

Tài liệu kỹ thuật dành cho Admin / IT: cài đặt webapp, bot Telegram, cấu hình môi trường, deploy production và vận hành hằng ngày.

Next.js 16 + FastAPI SQLite / PostgreSQL aiogram 3.x Docker · VPS · Railway
🏛️

Kiến trúc hệ thống

5 thành phần — hiểu trước khi cài
🌐 FrontendNext.js 16 · port 3001 (dev) / 3000 (docker)
⚡ Backend APIFastAPI · port 8000 · /docs = Swagger
🔄 Workerpython -m app.worker · lead scoring, tự động hóa CSKH, báo cáo BOD
🤖 Telegram Botpython -m bot.main · aiogram polling
🗄️ DatabaseSQLite (backend/jama.db) mặc định · PostgreSQL khi chạy Docker
Luồng dữ liệu: Frontend gọi Backend qua NEXT_PUBLIC_API_URL. Bot gọi Backend qua API_BASE_URL. Worker và Backend dùng chung database. Backend push thông báo Telegram qua Bot API (token), Bot nhận lệnh qua polling — 2 chiều độc lập, cùng 1 token.

📦 Yêu cầu hệ thống

Python3.11+ (backend, bot, worker)
Node.js18+ (khuyến nghị 20+, frontend)
RAMTối thiểu 2GB · khuyến nghị 4GB (VPS)
OSWindows 10/11 (dev) · Ubuntu 22.04/Debian 12 (VPS)

🗄️ SQLite hay PostgreSQL?

SQLite (mặc định)Zero-config, 1 file backend/jama.db — đủ cho 100 nhân sự, backup = copy file
PostgreSQLKhi chạy docker-compose — chịu tải cao hơn, nhiều kết nối đồng thời

Chuyển đổi chỉ cần đổi DATABASE_URL — bảng tự tạo khi backend khởi động (kèm seed data).

Cài trên Windows (dev / chạy nội bộ)

Cách nhanh nhất: double-click start.cmd

⚡ Cách 1 — start.cmd (khuyến nghị)

  1. Clone repo
    git clone https://github.com/anhthien8/jama-crm.git
cd jama-crm
  2. Cài dependencies lần đầu
    # Backend
cd backend
pip install -r requirements.txt

# Frontend
cd ../frontend
npm install
  3. Tạo file cấu hình — copy .env.example → backend/.env, bắt buộc điền JWT_SECRET_KEY (xem tab 🔐). Frontend đã có sẵn frontend/.env.local trỏ về http://localhost:8000/api/v1.
  4. Double-click start.cmd — script tự dừng process cũ trên port 8000/3001, khởi động Backend + Frontend, mở trình duyệt tại http://localhost:3001.
  5. Chạy Worker + Bot (2 cửa sổ cmd riêng, nếu cần tự động hóa & Telegram):
    # Cửa sổ 1 — Worker (tự động hóa CSKH, báo cáo BOD)
cd backend && python -m app.worker

# Cửa sổ 2 — Telegram Bot
cd telegram-bot && pip install -r requirements.txt && python -m bot.main
Địa chỉ sau khi chạy: Frontend http://localhost:3001 · Backend http://localhost:8000 · API Docs http://localhost:8000/docs · Health http://localhost:8000/health

Deploy Production — Docker / VPS / Railway

Chọn 1 trong 3 con đường

🚀 Cách 1 — VPS 1 lệnh (khuyến nghị cho production)

Script tự cài Docker, Nginx reverse proxy, SSL Let's Encrypt, firewall UFW, Fail2ban và deploy toàn bộ 5 service:

# SSH vào VPS (Ubuntu 22.04 / Debian 12, RAM ≥ 2GB) rồi chạy:
curl -sL https://raw.githubusercontent.com/anhthien8/jamaerp/main/deploy/setup.sh | sudo bash -s your-domain.com your-email@gmail.com

App được cài tại /opt/jama-crm. Sau khi xong, mở https://your-domain.com.

Việc thường dùngLệnh
Cập nhật phiên bản mớicd /opt/jama-crm/deploy && bash quick-deploy.sh (git pull + rebuild + health check)
Restart sau khi sửa .envcd /opt/jama-crm && ./restart.sh
Xem log 1 service./logs.sh backend (hoặc frontend / telegram-bot / worker)

🐳 Cách 2 — Docker Compose thủ công

  1. Copy .env.example → .env ở thư mục gốc, điền đủ biến (tab 🔐)
  2. Chạy:
    docker compose up -d --build
  3. 5 container tự lên: postgres, redis, backend, frontend, telegram-bot, worker. Kiểm tra: docker compose ps và curl localhost:8000/health
Lưu ý: bản Docker dùng PostgreSQL (không phải SQLite) — dữ liệu nằm trong volume postgres_data.

☁️ Cách 3 — Railway (đang dùng)

  1. Kết nối repo GitHub với Railway — mỗi lần git push origin main là tự deploy
  2. Tab Variables của từng service: thêm JWT_SECRET_KEY, TELEGRAM_BOT_TOKEN, LLM_API_KEY, CORS_ORIGINS (domain frontend)
  3. Frontend service cần NEXT_PUBLIC_API_URL=https://[backend-domain]/api/v1
  4. Kiểm tra health: https://jamacrm-production.up.railway.app/health
SQLite trên Railway: filesystem không bền vững giữa các lần deploy — gắn Volume cho file jama.db hoặc chuyển sang PostgreSQL plugin của Railway.

Setup Bot Telegram

Từ tạo bot đến briefing nhóm — 15 phút

Bước 1-3: Tạo bot & cấu hình token

  1. Tạo bot với @BotFather — Telegram → tìm @BotFather → /newbot → đặt tên JAMA HOME Trợ lý → username kết thúc bằng bot (vd jamahome_crm_bot) → nhận token.
  2. Điền token vào TELEGRAM_BOT_TOKEN ở env của backend (để push thông báo) và telegram-bot (để nhận lệnh) → restart 2 service.
  3. Giữ Group Privacy = ENABLED (mặc định của BotFather) — trong nhóm bot chỉ đọc lệnh /, không đọc trò chuyện của nhân viên. Kiểm tra: /mybots → Bot Settings → Group Privacy.

Bước 4-6: Nhóm công ty

  1. Tạo nhóm JAMA HOME — Công việc, bật Topics (Edit nhóm → Topics), thêm nhân viên + thêm bot (nên cho bot làm admin nhóm)
  2. Lấy Chat ID — gõ /id trong nhóm → bot trả số âm dạng -100xxxx
  3. Dán vào CRM — Cài đặt → Tự động hóa → ô 📢 Nhóm Telegram công ty → Lưu → bấm Gửi briefing nhóm để test

Topics khuyến nghị trong nhóm

📢 BriefingBot đăng briefing 8h sáng (khóa, chỉ admin+bot)
🏗️ Báo cáo công trình/baocao + ảnh
📦 Vật tư & Duyệt/vatlieu + nút duyệt inline
🚨 Sự cố/suco — không bị trôi
⏰ Chấm công/checkin · /checkout
💬 Trao đổiChat tự do

Bước 7: Liên kết nhân viên (bắt buộc để dùng lệnh)

  1. Nhân viên chat riêng với bot → gõ /id → nhận User ID (số dương)
  2. Admin mở CRM → /hr → hồ sơ nhân viên → điền Telegram User ID
  3. Nhân viên gõ /start → bot chào đúng tên + role là thành công
Cơ chế xác thực: bot map telegram_user_id → tài khoản CRM → cấp JWT — nhân viên chỉ thấy đúng dữ liệu theo role của mình, kể cả qua bot.

Checklist cấu hình sau cài đặt

Làm 1 lần bằng tài khoản admin, theo thứ tự
  1. Đổi toàn bộ mật khẩu mặc định — /hr → từng tài khoản seed → đặt mật khẩu mới. Xóa/khóa tài khoản demo không dùng.
  2. Cấu hình AI Model — Cài đặt → 🧠 Cấu hình AI Model: chọn model free (Groq khuyến nghị — lấy key tại console.groq.com), dán key, đặt model dự phòng khác nhà cung cấp (vd Gemini — aistudio.google.com) → bấm 🔌 Test kết nối → Lưu. Áp dụng ngay cho 5 AI agents + bot, không cần restart.
  3. Cấu hình Tự động hóa — Cài đặt → 🤖 Tự động hóa: ngưỡng nhắc CSKH (mặc định 3 ngày), thu hồi lead (7 ngày), giờ gửi báo cáo BOD (8h). Bấm ▶️ Chạy ngay để test.
  4. Kết nối nhóm Telegram — dán Chat ID nhóm (tab 🤖 Bot Telegram) → test 📢 Gửi briefing nhóm.
  5. Bật sao lưu tự động — Cài đặt → 💾 Sao lưu: mặc định 5h sáng, giữ 180 ngày. Kết nối Google Drive (OAuth) để có bản backup ngoài server → bấm 💾 Sao lưu ngay để test.
  6. Tạo tài khoản nhân viên thật — /hr → + Nhân viên → chọn đúng role (role quyết định toàn bộ quyền truy cập) → điền Telegram User ID.
  7. Kiểm tra worker đang chạy — xem log worker có dòng Health check | status=healthy mỗi 30 giây. Không có worker = không có nhắc CSKH / báo cáo / backup tự động.
  8. Gửi link hướng dẫn cho nhân viên — https://[domain]/huong-dan.html (đăng vào topic 📢 của nhóm).
