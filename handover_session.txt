## 1. Summary of Completed Features

### Feature A: Project Pipeline & Operational Workflow (Kanban + Notes + Media)
- **Database Schema Upgrades (`backend/app/models/project.py`):**
  - Added `stage` column (`design`, `quotation`, `procurement`, `construction`, `acceptance`, `completed`) to `Project` model.
  - Added `stage` and `final_file_url` columns to `Task` model.
  - Created `TaskActivity` model to track chronological updates/notes per task (with text content and optional `media_url` for images).
- **Auto-Conversion Hook (`backend/app/api/leads.py`):**
  - Updated the lead update stage API. When a lead's stage is transitioned to `signed_design` (Đã ký thiết kế), the backend automatically creates a `Customer` (by matching lead phone/id), a `Project`, and seeds 19 standard tasks corresponding to the operational stages.
- **Backend API Endpoints (`backend/app/api/projects.py`):**
  - `GET /projects/pipeline/kanban` - Returns projects grouped by stage.
  - `PUT /projects/{project_id}/stage` - Updates project stage.
  - `GET /projects/tasks/{task_id}/activities` - Lists task comments/timeline.
  - `POST /projects/tasks/{task_id}/activities` - Adds task notes/media attachment (base64 supported).
  - `PUT /projects/tasks/{task_id}/final-file` - Saves final delivery document link per task stage.
  - `PUT /projects/tasks/{task_id}/status` - Updates task status (`todo`, `in_progress`, `done`) and auto-recalculates total project progress %.
- **Frontend Upgrades (`frontend/src/app/projects/page.tsx`):**
  - Toggle between **📋 Danh sách** and **📊 Pipeline Kanban** view (Kanban default).
  - Grouped project tasks in Project Details Modal by their operational stage (Thiết kế, Báo giá, Thu mua, Thi công, Nghiệm thu).
  - Designed Task Details Modal to allow updating status, saving Final File, and uploading notes with images/attachments via a chronological activity feed.

### Feature B: Inventory Access Control (Kho vật tư)
- **Role Permission Definitions (`frontend/src/lib/roles.ts`):**
  - Added `'purchasing'` (Nhân viên Thu mua) to the `UserRole` union.
  - Configured `ROLE_PERMISSIONS.purchasing` to allow inventory viewing and contracts viewing.
  - Disabled `canViewInventory` for `leader` and `accountant` roles. Sidebar automatically filters and hides "/inventory" link for unauthorized roles.
- **Backend API Enforcement (`backend/app/api/inventory.py`):**
  - Added `verify_inventory_access` dependency to block non-admin and non-purchasing requests with HTTP 403 Forbidden.
